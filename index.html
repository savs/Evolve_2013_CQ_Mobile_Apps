Let's talk about mobile

Mobile challenges

You are here

You have a big investment in your existing website:

  • Lots of content
  • Lots of process
  • Lots of integrations

process = editorial processs; workflows for review

You need to address mobile

  • Mobile websites are easy (with CQ)
  • Mobile apps are hard

Mobile websites are easy because:

  • live copy/multi-site manager
  • emulators and simulators
  • device groups and detection

Mobile apps are hard

  • Using your content (without abandoning existing processes)
  • Providing back-end server infrastructure (without abandoning existing integrations)
  • Developing for mobile platforms (installing every SDK and writing in every programming language)

Back-end infrastructure: need to be able to script, store, scale

Mobile experiences are hard

  • unique functionality (touch, sensors)
  • unique constraints (memory, performance, battery, connectivity)
  • uniquely difficult to update

updates are tough:

  • going to an agency
  • pushing to app store(s)
  • getting users to update

poor experience impacts usage as well as brand

Key challenges

  • Repurposing content and workflows from CQ
  • Leveraging existing infrastructure to feed complex apps
  • Empowering Marketing / Giving control back
