Let's talk about mobile.

You are here

You have a big investment in your existing website:

  • Lots of content
  • Lots of processes
  • Lots of training
  • Lots of integrations
  • process = editorial processes; workflows for review, defined roles/responsibilities
  • training = staff time invested in knowing authoring tools
  • integration = ecommerce, crm, authentication/authorisation, enterprise data

You need to address mobile

Mobile challenges

  • Mobile websites are easy (with CQ)
  • Mobile apps are hard

Builds: mobile websites; mobile apps.

Mobile websites are easy because:

  • live copy/multi-site manager
  • device groups and detection
  • emulators and simulators
  • responsive design and adaptive images

Mobile apps are hard

How do you:

  • Re-use your content
  • Provide back-end server infrastructure
  • Develop for multiple mobile platforms
  • Content: without abandoning existing processes
  • Infrastructure: without abandoning existing integrations, and scale
  • Platforms: without installing every SDK and learning every language

Mobile experiences are hard

Unique challenges:

  • New functionality to use (touch, sensors)
  • Constraints (memory, performance, battery, connectivity)
  • Difficult to update

updates are tough:

  • going to an agency
  • pushing to app store(s)
  • getting users to update

poor experience impacts usage as well as brand

Key challenges

  • Repurposing content and workflows from CQ
  • Leveraging existing infrastructure to feed complex apps
  • Empowering Marketing and giving control back

CQ Architecture

  • OSGi Framework
  • Java, JSP, JS
  • Content store
  • Scalable
  • OSGi great modular framework for bundles and applications
  • Common server-side languages supported
  • content store based on java content repository open standard
  • already trust it to deliver high-traffic websites

Scenario: BlueSky

The requirement

  • Custom mobile application
  • Re-use server architecture
  • Deliver a dynamic experience

Builds: custom app; server; dynamic

  • app: app store discoverability
  • server: keep investment, leverage desktop
  • dynamic: tailored to mobile, mobile features

What we built

  • Third-party client app
    • PhoneGap, Angular.js
  • Server-side JSPs emitting JSON
  • Hypermedia API
  • third-party; only interaction was thru API spec
  • server-side: re-use existing content; easy to develop with CQ skills
  • api: only known endpoint was the first one.

Demo

  • Geometrixx Outdoors is a great demo ecommerce site
  • Geometrixx Outdoors has a communities section, demoing social communities
  • Communities are rich and engaging with a ton of content
  • ... but large amount of content doesn't work well as a mobile experience, so it's not included in mobile site
  • BlueSky app provides a community-centric mobile experience for Geo Outdoors
  • App - login, communities
  • Add new article - GPS, camera
  • Settings, invite - addressbook

The value

  • Mobile experience, not mobile effort
  • CQ as app server leverages existing investment, common technologies
  • Loosely-coupled API provides flexibility, minimises pain
